# Gebo - Active Context

## 1. Visión del Negocio
Gebo es una plataforma logística de bajo costo en Uruguay enfocada en la gestión y despacho de choferes. El modelo principal consiste en que el cliente contrata un chofer profesional para que maneje el vehículo del propio cliente. 
Para sostener la red logística, la empresa mantiene una flota propia de "vagonetas" que se encargan del despliegue (llevar al chofer hacia el vehículo del cliente) y del rescate automatizado (recoger al chofer cuando termina su servicio).

## 2. Arquitectura Técnica
- **Base de Datos:** Supabase (PostgreSQL 15+).
- **Geoespacial:** PostGIS integrado con indexación geométrica H3 de Uber (Resolución 8, ~700m de radio).
- **Backend:** Python (FastAPI + lógica adaptadora `h3-py`).
- **Frontend:** React (Vite) + Leaflet (mapas nativos) + Supabase Realtime (WebSockets `postgres_changes`).

## 3. Hitos Tecnológicos Implementados
- **Triggers de Rescate Automático (PL/pgSQL):** Al cambiar el estado de una tabla `faenas` a `finalizada`, un disparador (`trg_faena_finalizada_rescate`) utiliza el operador `<->` de PostGIS para localizar la vagoneta `disponible` más cercana e inserta de forma transaccional un `traslado_equipo` de tipo `retorno`.
- **Indexación Espacial H3 (Alta Escala):** Las tablas `posiciones` y `faenas` autocalculan el índice H3 (`h3_res8`) al insertarse/actualizarse sus coordenadas geográficas, utilizando la extensión de Supabase o un fallback funcional puro de PL/pgSQL.
- **Radar de Escasez en Tiempo Real:** El `AdminDashboard` renderiza zonas calientes H3 de forma matemática y nativa sobre Leaflet. Evalúa cruces de oferta/demanda en un anillo de vecindad $k=1$ (radio de ~1500m con Haversine) para pintar hexágonos rojos donde se acumulan clientes sin vagonetas disponibles.
- **Validación E2E:** Flujo espacial validado mediante el script automatizado en Python `test_integrado_h3.py`.
- **Ruteo Dinámico Continuo e Interceptación en Ruta (Micro-VRP):** Se validó exitosamente la interceptación de choferes en calle. La base de datos realiza la re-secuenciación automática (`secuencia = 1` y `secuencia = 2`) priorizando la proximidad. El motor en Python (`routing_engine.py`) utiliza la fórmula Haversine para estimar ETAs y recálculos acumulados, logrando una reducción del ~35% de kilometraje de flota estimado mediante re-secuenciación en tiempo real.
